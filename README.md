# مدرّب القواعد والكتابة للآيلتس — IELTS Grammar & Writing Trainer

تطبيق محلي بالكامل (offline-first) لمستخدم واحد، لتمرين يومي قصير (١٥–٢٠ دقيقة) على أنماط أخطاء ثابتة بين جلسات المدرّب البشري.

- **لا خدمات مدفوعة، لا مفاتيح API، لا خادم، لا حساب.** كل شيء يعمل في المتصفّح دون إنترنت.
- كل التمارين (قواعد + معرفة صوتية) **حتمية قائمة على القواعد** — لا أي ذكاء اصطناعي.
- التطبيق يبني **الخريطة الصوتية (طبقة المعرفة)** فقط: النبر، رموز IPA، الشوا — **بلا أي صوت** (لا ميكروفون، لا تسجيل، لا تقييم نطق؛ ذلك يبقى في ELSA Speak).
- فحص الكتابة **لا يتصل بأي خدمة**: يُنشئ موجّه تقييم جاهزاً تنسخه إلى محادثة Claude لديك.

---

## كيف تشغّله

التطبيق صفحة واحدة بـ HTML/CSS/JS صِرف. لا حاجة لأي بناء أو تثبيت.

**الطريقة الأبسط:** افتح ملف `index.html` مباشرةً بالنقر المزدوج عليه في المتصفّح (Chrome/Edge/Firefox). يعمل من `file://` لأن البيانات تُحمَّل عبر وسوم `<script>` لا عبر `fetch`.

**(بديل) عبر خادم محلي بسيط** إن رغبت:
```bash
cd ielts-trainer
python -m http.server 8000
# ثم افتح http://localhost:8000
```

التقدّم يُحفظ تلقائياً في `localStorage` للمتصفّح. زر **«تصدير بياناتي (JSON)»** في الأسفل يحفظ نسخة احتياطية تشاركها مع مدرّبك.

## المزايا (MVP)

1. **جلسة ١٥ دقيقة** — ١٦ عنصراً مختلطاً (قواعد + صوتيات) موزونة نحو نقاط ضعفك (الأقل دقّة يأخذ نصيباً أكبر)، وتنتهي بملخّص.
2. **التمارين** — تدرّب على قاعدة بعينها أو خليطاً من الكل.
3. **لوحة التقدّم** — لكل قاعدة على حدة: الدقّة ٪، عدد المحاولات، التتابع، آخر تدريب. لا توجد «درجة إجمالية».
4. **القاموس الصوتي** — بطاقات نطق (IPA + تقريب عربي + نمط النبر + سياق مالي)، قابلة للتوسيع.
5. **فحص الكتابة** — يولّد موجّه تقييم بمعايير الآيلتس الأربعة + وسم كل خطأ بنمط المراقبة + تنبيه انحياز موضوع المالية/CMA.

### الآلية الجوهرية: «فكّر قبل أن تكشف»
كل عنصر يطلب منك: (١) إجابتك، ثم (٢) **سطر تبرير إلزامي** قبل أن يُكشف الصواب. يُحفظ كلٌّ من إجابتك وتبريرك لمراجعة أنماط تفكيرك لاحقاً (ولمدرّبك).

## أنماط الأخطاء المستهدفة (تُتابَع كلٌّ منها منفصلاً)

| المعرّف | القاعدة |
|---|---|
| `articles` | أدوات التعريف والتنكير (a/an/the/بدون) |
| `verbform` | صيغة الفعل بعد حروف الجر وضمائر الوصل |
| `copula` | فعل الكون والمطابقة مع المفرد الغائب |
| `vocab` | مفردات معكوسة المعنى (يبدأ بـ employer/employee) |
| `stress` `sentence_stress` `ipa` `schwa` | الصوتيات (معرفة النطق) |
| `dictionary` | قاموس النطق الشخصي (مراجعة فقط) |

## أين تعيش البيانات وكيف تعدّلها

كل بنوك العناصر ملفات قابلة للقراءة والتحرير يدوياً داخل مجلّد `data/`:

```
data/grammar_articles.js     قاعدة الأدوات
data/grammar_verbform.js     صيغة الفعل
data/grammar_copula.js       فعل الكون والمطابقة
data/grammar_vocab.js        المفردات معكوسة المعنى
data/phonetics_stress.js     نبر الكلمة
data/phonetics_sentence.js   نبر الجملة
data/phonetics_ipa.js        رموز IPA
data/phonetics_schwa.js      الشوا
data/phonetics_dictionary.js قاموس النطق
```
> ملاحظة: المحتوى داخل كل ملف **بنية JSON خالصة** ملفوفة بسطر `window.IELTS_DATA.<rule> = [ ... ];`. هذا يسمح بفتح التطبيق مباشرةً من `file://` دون خادم (لو كانت ملفات `.json` صرفة لمنعها المتصفّح بسبب CORS). حرِّر داخل القوسين `[ ... ]` فقط.

### إضافة عنصر قواعد (اختيار من متعدد) — مثال `vocab`:
```js
{ id: "voc24", rule: "vocab", type: "mc",
  sentence_with_gap: "I will ___ the report tomorrow.",
  correct_answer: "complete", distractors: ["compete"],
  explanation: "complete = يُكمل، compete = ينافس. هنا إكمال التقرير → complete." }
```

### إضافة عنصر «املأ الفراغ» — `verbform` / `copula`:
```js
{ id: "vf23", rule: "verbform", type: "fill",
  sentence_with_gap: "She is keen on ___ new skills.", base_verb: "learn",
  correct_answer: "learning", accept: ["learning"],
  explanation: "keen on + gerund." }
```
- `___` تمثّل الفراغ. `accept` قائمة الإجابات المقبولة (تجاهل حالة الأحرف والمسافات).

### إضافة زوج مفردات مربك جديد:
أضِف عنصرَين (واحد لكل اتجاه) في `data/grammar_vocab.js` بنفس صيغة `type: "mc"` أعلاه. القائمة مصمّمة لتوسّعها.

### إضافة عنصر نبر كلمة:
```js
{ id: "st21", rule: "stress", type: "stress", word: "investment",
  syllables: ["in", "VEST", "ment"], correct_index: 1,
  explanation: "in-VEST-ment: النبر على الثاني." }
```

### إضافة كلمة للقاموس:
إمّا من داخل التطبيق (تُحفظ في المتصفّح)، أو دائمياً في `data/phonetics_dictionary.js`:
```js
{ word: "amortization", ipa: "/əˌmɔː.taɪˈzeɪ.ʃən/", arabic_approximation: "أَمورتايزايشِن",
  stress_pattern: "a-mor-ti-ZA-tion", finance_context: "Amortization spreads the cost of an intangible asset over time." }
```

> بعد تعديل أي ملف بيانات: أعِد تحميل الصفحة (F5). تأكّد أن `id` فريد، وأن الفاصلة بين العناصر صحيحة.

## أين يُحفظ تقدّمي

في `localStorage` تحت المفتاح `ielts_trainer_v1`، ويشمل: إحصاءات كل قاعدة، وسجلّ كامل بكل محاولة (الإجابة + التبرير + الصواب/الخطأ + الوقت)، وإضافاتك للقاموس، وتتابع الأيام. استخدم زر **التصدير** لأخذ نسخة JSON.

لمسح كل شيء: امسح بيانات الموقع من المتصفّح، أو نفّذ في وحدة التحكّم:
```js
localStorage.removeItem("ielts_trainer_v1"); location.reload();
```
